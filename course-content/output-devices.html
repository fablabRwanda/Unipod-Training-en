Overview

Welcome to Output Devices! This week, you'll learn to control actuators and displays—the "hands and voice" of your projects. From LEDs and motors to relays and displays, outputs let your microcontroller interact with the physical world.

In African contexts, output devices enable powerful applications: automated irrigation systems for agriculture, safety alarms for security, educational displays for schools, and more. We'll explore different types of outputs, learn to drive them safely (especially high-power devices like motors), and create meaningful behaviors.

This week builds on Embedded Programming (where you learned to control outputs) and Input Devices (where you learned to read sensors). By the end of the week, you'll understand how to choose and control outputs for applications that solve real problems in your community.

Learning Outcomes

By the end of this week, you will be able to:

  • Understand different output types including LEDs, motors (DC, stepper, servo), relays, buzzers, displays, and their characteristics
  • Drive low-power outputs directly from microcontroller pins (LEDs, small buzzers)
  • Drive high-power outputs safely using transistors, motor drivers, and relays with proper protection circuits
  • Control outputs with PWM for variable speed, brightness, or position control
  • Create meaningful behaviors by combining outputs with sensor inputs and programming logic
  • Document output circuits including power requirements, protection components, and safety considerations

Core Concepts & Theory

1. Types of Output Devices

Output devices can be categorized by their function and power requirements:

  • Visual Outputs:
    • LEDs: Simple, low-power indicators. Can be driven directly from GPIO pins (with current-limiting resistor).
    • LED Strips: Multiple LEDs in series/parallel. Often require external power supply and PWM control.
    • LCD Displays: Text or graphics displays. Usually communicate via I2C or SPI.
    • OLED Displays: High-contrast displays. Often I2C or SPI.
  • Mechanical Outputs:
    • DC Motors: Simple rotation. Require motor driver (H-bridge) and external power supply.
    • Stepper Motors: Precise position control. Require stepper driver (like A4988, DRV8825).
    • Servo Motors: Position control via PWM. Can be driven directly from PWM-capable pins.
    • Solenoids: Linear actuation. Require high current, need transistor/relay driver.
  • Audio Outputs:
    • Buzzers: Simple beeps. Can be driven directly (piezo) or via transistor (active buzzer).
    • Speakers: Audio output. Require amplifier circuit.
  • Switching Outputs:
    • Relays: Switch high-voltage/high-current devices (lights, pumps, appliances). Require transistor driver.
    • Solid-State Relays (SSR): Electronic switching, faster than mechanical relays.

2. Driving Outputs Safely

Microcontroller pins can only supply limited current (typically 20-40mA). For higher-power devices, you need driver circuits:

  • Current-Limiting Resistors: For LEDs. Calculate: R = (VCC - V_LED) / I_LED. Example: (5V - 2V) / 20mA = 150Ω (use 220Ω standard).
  • Transistors (BJT or MOSFET): Switch higher currents. Use NPN BJT or N-channel MOSFET for low-side switching. Add base/gate resistor to limit current from microcontroller.
  • Motor Drivers: H-bridge circuits (L293D, L298N, TB6612FNG) allow bidirectional motor control and handle high currents.
  • Protection Diodes: Flyback diodes across motors/relays prevent voltage spikes from damaging circuits when devices turn off.
  • Optocouplers: Isolate high-voltage circuits from microcontroller for safety.

3. PWM (Pulse Width Modulation)

PWM allows you to control output intensity by rapidly switching between HIGH and LOW:

  • Duty Cycle: Percentage of time signal is HIGH (0% = off, 100% = fully on, 50% = half intensity).
  • Frequency: How fast the signal switches (typically 500Hz-20kHz). Too slow = visible flicker, too fast = inefficiency.
  • Applications:
    • LED brightness control (dimming)
    • Motor speed control
    • Servo position control (specific pulse widths correspond to angles)
  • In Arduino: Use analogWrite(pin, value) where value is 0-255 (8-bit PWM).

4. Power Considerations

Always consider power requirements:

  • Voltage: Match output device voltage to your power supply (3.3V, 5V, 12V, etc.).
  • Current: Ensure power supply can provide enough current. Motors can draw 100mA-2A+.
  • Separate Power Supplies: Use separate power for motors/relays and microcontroller. Connect grounds together.
  • Voltage Regulators: Step down higher voltages (12V → 5V) for microcontroller. Use capacitors for filtering.
  • Battery Considerations: Motors drain batteries quickly. Consider power management for portable projects.

Mini-Lab: Motor Control with Speed Control

Let's build a circuit to control a DC motor with variable speed—essential for applications like fans, pumps, or automated systems.

Step 1: Gather Components

  • Microcontroller board (Arduino Uno or your custom board)
  • Small DC motor (5-12V, low current for safety)
  • Motor driver (L293D or simple NPN transistor + diode)
  • Potentiometer (10kΩ) for speed control
  • Flyback diode (1N4007 or similar)
  • External power supply (9V battery or 12V adapter) for motor
  • Breadboard and jumper wires

Step 2: Wire the Circuit

  1. Motor Driver (L293D):
    • VCC1 (pin 16) → 5V (logic power)
    • VCC2 (pin 8) → External 9V/12V (motor power)
    • GND → Common ground (connect all grounds together!)
    • Input 1 (pin 2) → Digital pin 9 (PWM)
    • Input 2 (pin 7) → GND (for one direction)
    • Enable 1 (pin 1) → Digital pin 10
    • Output 1 (pin 3) → Motor terminal 1
    • Output 2 (pin 6) → Motor terminal 2
  2. Potentiometer:
    • One end → 5V
    • Other end → GND
    • Wiper (middle) → Analog pin A0
  3. Flyback Diode: Connect across motor terminals (cathode to positive, anode to negative) to protect circuit.

Step 3: Write the Code

// Pin definitions
#define MOTOR_PWM 9      // PWM pin for motor speed
#define MOTOR_ENABLE 10  // Enable pin
#define POT_PIN A0       // Potentiometer for speed control

void setup() {
  pinMode(MOTOR_PWM, OUTPUT);
  pinMode(MOTOR_ENABLE, OUTPUT);
  
  // Enable motor driver
  digitalWrite(MOTOR_ENABLE, HIGH);
  
  Serial.begin(9600);
  Serial.println("Motor Control with Speed Control");
}

void loop() {
  // Read potentiometer (0-1023)
  int potValue = analogRead(POT_PIN);
  
  // Convert to PWM value (0-255)
  int motorSpeed = map(potValue, 0, 1023, 0, 255);
  
  // Set motor speed
  analogWrite(MOTOR_PWM, motorSpeed);
  
  // Print speed percentage
  Serial.print("Motor Speed: ");
  Serial.print((motorSpeed * 100) / 255);
  Serial.println("%");
  
  delay(100);
}

Step 4: Test and Experiment

  1. Upload code and open Serial Monitor
  2. Turn potentiometer to see motor speed change
  3. Experiment: Add a button to reverse motor direction
  4. Experiment: Control motor based on sensor input (e.g., temperature sensor controls fan speed)

Tips for Success

  • Start with low-power devices: Begin with LEDs and buzzers before moving to motors and relays.
  • Always use protection: Add flyback diodes across motors/relays to prevent voltage spikes.
  • Check power requirements: Ensure your power supply can provide enough current. Motors can draw much more than you expect!
  • Use separate power supplies: Power motors/relays separately from microcontroller. Connect grounds together.
  • Test incrementally: Test output device with simple code first, then add complexity (sensor integration, PWM control).
  • Read datasheets: Check output device datasheet for voltage, current, and control requirements.
  • Start with low speed/voltage: When testing motors, start slow and increase gradually to avoid damage.
  • Document everything: Record power consumption, component choices, and any issues you encounter.
