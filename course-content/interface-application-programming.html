Overview — Interface and Application Programming

Welcome to Interface and Application Programming! This week, you'll learn to build software interfaces that communicate with your hardware projects. From simple desktop applications to web-based dashboards, interfaces make your projects accessible and user-friendly.

We'll explore different approaches (Processing, Python, web-based interfaces) and choose tools that work well in African contexts—considering offline capabilities, low-spec devices, and shared lab computers. By the end of the week, you'll have created an interface that reads sensor data, controls outputs, or visualizes information from your projects.

Learning Objectives

  • Send and receive data between a program and a microcontroller.
  • Create basic user interfaces (desktop, web, or mobile) to visualize or control hardware.
  • Choose languages and frameworks that fit your lab’s skills and connectivity.

Assignments

  • Individual: Build an application that reads from or writes to your board (e.g. sliders controlling LEDs, graphs plotting sensor data).
  • Group: Compare at least two approaches (e.g. Processing vs. Python vs. web) and document pros/cons for your UNIPOD.

Documentation & Evidence

  • Source code of your interface and screenshots of it in use.
  • Explanation of the communication protocol (serial, WebSocket, HTTP, etc.).
  • Reflection on deployment constraints (offline-first, low-spec devices, shared lab computers).
