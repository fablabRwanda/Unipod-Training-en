Overview

Welcome to Interface and Application Programming! This week, you'll learn to build software interfaces that communicate with your hardware projects. From simple desktop applications to web-based dashboards, interfaces make your projects accessible and user-friendly.

In African contexts, interfaces enable powerful applications: monitoring systems for agriculture, data visualization for health systems, control panels for water management, and more. We'll explore different approaches (Processing, Python, web-based interfaces) and choose tools that work well in African contexts—considering offline capabilities, low-spec devices, and shared lab computers.

Core Concepts & Theory

1. Communication Protocols

Interfaces communicate with hardware using different protocols:

  • Serial Communication (UART):
    • Simple, direct connection between computer and microcontroller
    • Baud rate: 9600, 115200, etc. (must match on both sides)
    • Use: Desktop applications, simple data logging
    • In Processing: Serial library
    • In Python: pyserial library
    • In JavaScript: SerialPort library (Node.js) or Web Serial API (browsers)
  • WebSocket:
    • Real-time bidirectional communication over network
    • Use: Web interfaces, remote monitoring
    • Requires: Server (ESP32, computer, or cloud)
  • HTTP/REST API:
    • Request-response protocol, works over network
    • Use: Web interfaces, mobile apps, cloud integration
    • Requires: Server (ESP32 web server, computer, or cloud)

2. Interface Approaches

Choose the right tool for your context:

  • Processing:
    • Visual programming language, great for graphics and animations
    • Pros: Simple, visual, good for learning, works offline
    • Cons: Less suitable for complex applications
    • Use: Data visualization, interactive art, simple control panels
    • Best for: Beginners, visual projects
  • Python (Tkinter, PyQt, Kivy):
    • Powerful, versatile, widely used
    • Pros: Lots of libraries, cross-platform, works offline
    • Cons: Requires Python installation
    • Use: Desktop applications, data analysis, complex interfaces
    • Best for: Intermediate users, data-heavy applications
  • Web-based (HTML/CSS/JavaScript):
    • Runs in browsers, accessible from any device
    • Pros: No installation needed, works on phones/tablets, easy to share
    • Cons: Requires server (ESP32, computer, or cloud), needs connectivity for remote access
    • Use: Dashboards, remote monitoring, mobile-friendly interfaces
    • Best for: When you need accessibility across devices
  • Node.js/Electron:
    • JavaScript-based desktop applications
    • Pros: Web technologies in desktop app, cross-platform
    • Cons: Larger file size, more complex setup
    • Use: Professional desktop applications

3. Designing for African Contexts

Consider these constraints when designing interfaces:

  • Offline-First:
    • Design interfaces that work without internet
    • Use local servers (ESP32, local computer) instead of cloud
    • Cache data locally for offline viewing
  • Low-Spec Devices:
    • Keep interfaces lightweight (minimal JavaScript, simple graphics)
    • Test on older computers or low-end devices
    • Avoid heavy frameworks or libraries
  • Shared Lab Computers:
    • Design interfaces that don't require installation (web-based or portable apps)
    • Save settings/data to user folders, not system folders
    • Make interfaces easy to reset/clear for next user
  • Limited Connectivity:
    • Minimize data transfer (send only necessary data)
    • Use efficient protocols (WebSocket instead of polling HTTP)
    • Compress data if sending large amounts

4. Data Visualization

Effective interfaces visualize data clearly:

  • Real-time Graphs: Show sensor data over time (temperature, light, etc.)
  • Gauges/Meters: Display current values (speed, percentage, etc.)
  • Status Indicators: LEDs, colors, icons showing system state
  • Tables/Lists: Organized data display
  • Maps: Geographic data visualization (for distributed sensor networks)

Mini-Lab: Python Interface for Sensor Monitoring

Let's create a simple Python interface that reads sensor data from Arduino and displays it in real-time—perfect for monitoring applications.

Step 1: Arduino Code (Sensor Reader)

// Arduino code - sends sensor data via serial
#define TEMP_PIN A0
#define LIGHT_PIN A1

void setup() {
  Serial.begin(9600);
  Serial.println("Sensor Monitor Ready");
}

void loop() {
  // Read sensors
  int tempRaw = analogRead(TEMP_PIN);
  int lightRaw = analogRead(LIGHT_PIN);
  
  // Convert and format data
  float temp = (tempRaw / 1024.0) * 5.0 * 100 - 50; // TMP36 conversion
  float light = (lightRaw / 1024.0) * 100; // Percentage
  
  // Send as CSV: temperature,light
  Serial.print(temp);
  Serial.print(",");
  Serial.println(light);
  
  delay(1000); // Send every second
}

Step 2: Python Interface Code

# Python interface using Tkinter and pyserial
import serial
import tkinter as tk
from matplotlib.figure import Figure
from matplotlib.backends.backend_tkagg import FigureCanvasTkAgg
import time

class SensorMonitor:
    def __init__(self, root):
        self.root = root
        self.root.title("Sensor Monitor")
        
        # Open serial port (change 'COM3' to your port)
        self.ser = serial.Serial('COM3', 9600, timeout=1)
        time.sleep(2)  # Wait for Arduino to reset
        
        # Create GUI
        self.setup_gui()
        
        # Start data reading
        self.update_data()
    
    def setup_gui(self):
        # Labels for current values
        self.temp_label = tk.Label(self.root, text="Temperature: --°C", font=("Arial", 16))
        self.temp_label.pack(pady=10)
        
        self.light_label = tk.Label(self.root, text="Light: --%", font=("Arial", 16))
        self.light_label.pack(pady=10)
        
        # Graph for temperature over time
        self.fig = Figure(figsize=(6, 4))
        self.ax = self.fig.add_subplot(111)
        self.ax.set_title("Temperature Over Time")
        self.ax.set_xlabel("Time (seconds)")
        self.ax.set_ylabel("Temperature (°C)")
        
        self.canvas = FigureCanvasTkAgg(self.fig, self.root)
        self.canvas.get_tk_widget().pack()
        
        # Data storage
        self.times = []
        self.temps = []
        self.max_points = 50  # Show last 50 points
    
    def update_data(self):
        if self.ser.in_waiting:
            try:
                line = self.ser.readline().decode('utf-8').strip()
                if ',' in line:
                    temp, light = map(float, line.split(','))
                    
                    # Update labels
                    self.temp_label.config(text=f"Temperature: {temp:.1f}°C")
                    self.light_label.config(text=f"Light: {light:.1f}%")
                    
                    # Update graph
                    current_time = len(self.times)
                    self.times.append(current_time)
                    self.temps.append(temp)
                    
                    # Keep only last N points
                    if len(self.times) > self.max_points:
                        self.times = self.times[-self.max_points:]
                        self.temps = self.temps[-self.max_points:]
                    
                    # Redraw graph
                    self.ax.clear()
                    self.ax.plot(self.times, self.temps, 'b-')
                    self.ax.set_title("Temperature Over Time")
                    self.ax.set_xlabel("Time (seconds)")
                    self.ax.set_ylabel("Temperature (°C)")
                    self.canvas.draw()
            except:
                pass  # Ignore errors
        
        # Schedule next update
        self.root.after(100, self.update_data)

# Run application
if __name__ == "__main__":
    root = tk.Tk()
    app = SensorMonitor(root)
    root.mainloop()

Step 3: Install Dependencies and Run

  1. Install Python libraries: 
    pip install pyserial matplotlib
  2. Upload Arduino code to your board
  3. Note your serial port (COM3 on Windows, /dev/ttyUSB0 on Linux, /dev/tty.usbserial on Mac)
  4. Update port in Python code
  5. Run Python script: python sensor_monitor.py

Recommended Software Tools

Processing (Recommended for Beginners)
  • Free and visual: Great for learning interface programming
  • Serial library built-in: Easy Arduino communication
  • Graphics-focused: Perfect for data visualization
  • Download: processing.org
Python (Recommended for Intermediate Users)
  • Powerful and versatile: Lots of libraries for everything
  • Libraries: pyserial (serial), matplotlib (graphs), tkinter (GUI built-in)
  • Cross-platform: Works on Windows, Mac, Linux
  • Install: pip install pyserial matplotlib
Web Technologies (HTML/CSS/JavaScript)
  • No installation: Runs in any browser
  • Accessible: Works on phones, tablets, computers
  • Libraries: Chart.js (graphs), Web Serial API (browser serial)
  • Use: When you need cross-device access
Node.js (For Advanced Users)
  • JavaScript everywhere: Same language for frontend and backend
  • Libraries: serialport, express, socket.io
  • Use: Professional applications, web servers

Tips for Success

  • Start simple: Begin with basic serial communication before adding complex features.
  • Test incrementally: Test communication first, then add GUI elements one at a time.
  • Handle errors gracefully: Add try/except blocks, check for connection, handle missing data.
  • Design for your context: Consider offline operation, low-spec devices, shared computers.
  • Use libraries: Don't reinvent the wheel—use existing libraries for graphs, GUIs, serial communication.
  • Document well: Comment your code, write installation instructions, explain data formats.
  • Test on target devices: Test on the actual computers/devices where it will be used.
  • Make it user-friendly: Clear labels, intuitive controls, helpful error messages.
