Overview

Welcome to Networking and Communications! This week, you'll learn to connect multiple devices together, enabling your projects to share data and work together. From simple serial communication to wireless networks, connectivity opens up powerful possibilities.

In African contexts, networking enables critical applications: distributed sensor networks for agriculture, remote monitoring for health systems, coordinated irrigation systems, and shared data displays. We'll explore wired protocols (UART, I2C, SPI) and wireless options (Wi-Fi, Bluetooth, LoRa where available).

This week builds on Input Devices and Output Devices (where you learned to read sensors and control actuators) and prepares you for Interface and Application Programming (where you'll create UIs that communicate with hardware). By the end of the week, you'll understand how to build networks of devices that can monitor environments, share data, and coordinate actions—essential for IoT applications in agriculture, health, and infrastructure monitoring.

Core Concepts & Theory

1. Wired Communication Protocols

Wired protocols are reliable and don't require radio hardware:

  • UART (Serial):
    • Simple two-wire communication (TX, RX)
    • Asynchronous (no clock signal needed)
    • Common baud rates: 9600, 115200
    • Use: Microcontroller-to-computer, simple board-to-board
    • In Arduino: Serial.begin(9600), Serial.print(), Serial.read()
  • I2C (Inter-Integrated Circuit):
    • Two-wire bus (SDA, SCL) for multiple devices
    • Each device has unique address (7-bit or 10-bit)
    • Master-slave architecture
    • Use: Connecting multiple sensors/displays to one microcontroller
    • In Arduino: Wire.begin(), Wire.beginTransmission(address), Wire.write()
  • SPI (Serial Peripheral Interface):
    • Four-wire bus (MOSI, MISO, SCK, SS/CS)
    • Faster than I2C, full-duplex
    • Master-slave with chip select for multiple devices
    • Use: High-speed communication with displays, SD cards, sensors
    • In Arduino: SPI.begin(), SPI.transfer()

2. Wireless Communication Options

Wireless enables remote monitoring and control:

  • Wi-Fi:
    • High data rate, internet connectivity
    • Requires access point/router
    • Modules: ESP32, ESP8266
    • Use: IoT projects, web-based interfaces, cloud connectivity
    • Power: Higher consumption, needs good power supply
  • Bluetooth:
    • Short-range (10-100m), low power
    • Direct device-to-device pairing
    • Modules: HC-05, HC-06, ESP32 (built-in)
    • Use: Mobile app connectivity, short-range sensor networks
    • Power: Lower than Wi-Fi, good for battery-powered devices
  • LoRa (Long Range):
    • Very long range (1-10km), very low power
    • Low data rate, good for sensor networks
    • Modules: SX1278, LoRaWAN modules
    • Use: Remote monitoring, agricultural sensor networks
    • Power: Very low, excellent for battery-powered devices

3. Network Topologies

How devices connect in a network:

  • Point-to-Point: Two devices directly connected (simplest, UART)
  • Bus: Multiple devices on shared bus (I2C, CAN bus)
  • Star: Central hub connects to multiple devices (Wi-Fi access point)
  • Mesh: Devices relay messages to each other (complex, requires routing)

4. Data Exchange Considerations

When designing communication:

  • Packet Format: Define structure (header, data, checksum)
  • Error Detection: Use checksums or CRC to detect corrupted data
  • Synchronization: Ensure sender and receiver are in sync (start/stop bits, framing)
  • Addressing: Identify which device should receive message (I2C addresses, IP addresses)
  • Flow Control: Prevent data loss when receiver is busy (handshaking, buffers)

Mini-Lab: Two-Board I2C Communication

Let's create a simple network where one board (master) reads a sensor and sends data to another board (slave) that displays it.

Step 1: Gather Components

  • Two microcontroller boards (Arduino Uno or your custom boards)
  • Temperature sensor (TMP36 or similar) for master board
  • LCD display (I2C) or LED for slave board
  • Two 4.7kΩ pull-up resistors (for I2C bus)
  • Jumper wires

Step 2: Wire the I2C Bus

  1. Connect SDA (data) lines together on both boards
  2. Connect SCL (clock) lines together on both boards
  3. Add 4.7kΩ pull-up resistors from SDA to 5V and SCL to 5V
  4. Connect all grounds together (common ground is essential!)
  5. Connect VCC together (both boards need power)

Step 3: Master Board Code (Sender)

// Master board - reads sensor and sends data
#include <Wire.h>

#define SLAVE_ADDRESS 0x08
#define TEMP_PIN A0

void setup() {
  Wire.begin();        // Master joins I2C bus
  Serial.begin(9600);
  Serial.println("Master: I2C Communication");
}

void loop() {
  // Read temperature sensor
  int tempRaw = analogRead(TEMP_PIN);
  float voltage = (tempRaw / 1024.0) * 5.0;
  float temperature = (voltage - 0.5) * 100;
  
  // Send data to slave
  Wire.beginTransmission(SLAVE_ADDRESS);
  Wire.write((byte)(temperature * 10));  // Send as integer (multiply by 10 for 0.1°C precision)
  Wire.endTransmission();
  
  Serial.print("Sent temperature: ");
  Serial.println(temperature);
  
  delay(1000);
}

Step 4: Slave Board Code (Receiver)

// Slave board - receives data and displays
#include <Wire.h>

#define SLAVE_ADDRESS 0x08

void setup() {
  Wire.begin(SLAVE_ADDRESS);  // Join I2C bus with address
  Wire.onReceive(receiveEvent); // Register event handler
  Serial.begin(9600);
  Serial.println("Slave: Ready to receive");
}

void loop() {
  delay(100);
}

// Function that executes when data is received
void receiveEvent(int howMany) {
  if (Wire.available()) {
    int tempValue = Wire.read();
    float temperature = tempValue / 10.0;  // Convert back to float
    
    Serial.print("Received temperature: ");
    Serial.println(temperature);
    
    // Here you could display on LCD, control LED, etc.
  }
}

Step 5: Test and Expand

  1. Upload master code to one board, slave code to other
  2. Open Serial Monitors for both boards
  3. Observe data being sent and received
  4. Experiment: Add more sensors to master, send multiple values
  5. Experiment: Add display to slave to show received data

Tips for Success

  • Start with wired: Master I2C or UART before attempting wireless—it's simpler and more reliable for learning.
  • Common ground is essential: All devices must share a common ground for communication to work.
  • Use pull-up resistors: I2C requires pull-up resistors (typically 4.7kΩ) on SDA and SCL lines.
  • Check addresses: For I2C, ensure each device has a unique address. Use I2C scanner sketch to find addresses.
  • Test incrementally: Test communication with simple data first (send "Hello"), then add complexity.
  • Document data format: Clearly define how data is structured (byte order, data types, packet format).
  • Consider power: Wireless modules consume more power. Plan power supply accordingly.
  • Test range: For wireless, test at different distances and document maximum reliable range.
