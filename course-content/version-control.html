Version Control

What is Git?

Git is a distributed version control system that tracks changes in your project files over time, enabling collaboration, history, branching, and safe experimentation.

What is GitHub?

GitHub is a cloud platform for hosting Git repositories. It adds collaboration features (issues, pull requests), access control, and deployment tools like GitHub Pages for static websites.

Install and Configure Git
  • git --version to confirm Git is installed
  • git config --global user.name "Your Name"
  • git config --global user.email "you@example.com"
Create SSH Keys and add to GitHub
  1. Generate a key: ssh-keygen -t ed25519 -C "you@example.com" (press Enter to accept defaults, set a passphrase if you want).
  2. Start the agent and add your key:
    eval "$(ssh-agent -s)"
    ssh-add ~/.ssh/id_ed25519
  3. Copy the public key:
    pbcopy < ~/.ssh/id_ed25519.pub (macOS)
  4. Add it in GitHub → Settings → SSH and GPG keys → New SSH key.
Basic Git Workflow
  1. Clone: git clone git@github.com:YOUR_USER/YOUR_REPO.git
  2. Create a branch: git switch -c feature/my-change
  3. Stage and commit: git add -A && git commit -m "Describe your change"
  4. Push: git push -u origin feature/my-change
  5. Open a Pull Request on GitHub and merge.
Host a Website with GitHub Pages
  1. Create a repository named YOUR_USER.github.io.
  2. Add your static site (HTML/CSS/JS) at the repo root.
  3. Commit and push to main.
  4. Your site will be available at https://YOUR_USER.github.io within a few minutes.
Deploy an Existing Project to GitHub Pages
  • Option 1 (user site): push your site to YOUR_USER.github.io.
  • Option 2 (project site): enable Pages in Repository Settings → Pages → set Source to main (or gh-pages) and folder to / or /docs.
Student Guide: Create Your Website
  1. Fork the template: documentation-page.
  2. Rename your fork to YOUR_USER.github.io for a user site, or keep the name for a project site.
  3. Clone your fork via SSH: git clone git@github.com:YOUR_USER/YOUR_USER.github.io.git
  4. Customize content in the repository (HTML files like index.html, week1.html, etc.).
  5. Commit and push changes: git add -A && git commit -m "Customize site" && git push
  6. Enable GitHub Pages in Settings if needed, then visit your site URL.
Tip: The template repository includes additional pages and partials to help you start fast. See the template at documentation-page.
