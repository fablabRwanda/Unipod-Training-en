Overview — Version Control & Documentation

Welcome to Version Control and Documentation! This week, you'll learn to use Git—a powerful tool for tracking changes in your work and collaborating with others. You'll also set up a digital portfolio to document your learning journey throughout the course.

Good documentation is essential for sharing knowledge across UNIPODs and building on each other's work. By the end of the week, you'll have a Git repository, a published website, and the skills to document your projects clearly so others can learn from and build on your work.

Learning Objectives

  • Install and configure Git on your laptop.
  • Create and clone a remote repository for your UNIPOD portfolio.
  • Publish a simple site for weekly documentation (HTML, MkDocs or similar).
  • Use branches, commits, and pull/push to track your work.

Assignments

  • Individual: Create a Git repository for your Fab/UNIPOD work and push at least one week of documentation.
  • Individual: Include a “who am I / what I want to make” page with text and images.
  • Group: As a lab, agree on a documentation style and folder structure so all students are consistent.

Documentation & Evidence

  • Link to your public repository and live site.
  • Screenshots of your Git history (commits), branch list, and remote configuration.
  • A short reflection on how version control will help your future projects.
