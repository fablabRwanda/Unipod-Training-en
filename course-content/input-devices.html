Overview

Welcome to Input Devices! This week, you'll learn to connect sensors to your microcontroller boards and read data from the physical world. Sensors are the "eyes and ears" of your projects—they measure light, temperature, distance, motion, and more.

In African contexts, sensors enable powerful applications: monitoring soil moisture for agriculture, detecting water quality for health, measuring energy consumption for efficiency. We'll explore different types of sensors (digital vs. analog), learn to interface them with microcontrollers, and visualize the data they produce.

This week builds on Embedded Programming (where you learned to read inputs) and prepares you for Output Devices (where you'll control actuators based on sensor readings). By the end of the week, you'll understand how to choose sensors for your projects and read reliable data even in challenging African environments (heat, dust, variable power).

Learning Outcomes

By the end of this week, you will be able to:

  • Understand sensor types including digital, analog, I2C, SPI, and their characteristics (range, accuracy, response time)
  • Interface sensors with microcontrollers using appropriate circuits (voltage dividers, pull-ups, amplifiers)
  • Read sensor data using digitalRead(), analogRead(), or communication protocols (I2C, SPI)
  • Calibrate sensors to account for environmental conditions and improve accuracy
  • Visualize sensor data using serial plots, basic UIs, or data logging
  • Document sensor behavior including noise, drift, and environmental sensitivity

Core Concepts & Theory

1. Digital vs. Analog Sensors

Sensors can be categorized by their output type:

  • Digital Sensors: Output HIGH/LOW signals. Examples: buttons, switches, PIR motion sensors, magnetic hall sensors.
    • Pros: Simple to read, immune to noise, fast response
    • Cons: Less information (just on/off)
    • Use: When you only need to detect presence/absence or threshold crossings
  • Analog Sensors: Output continuous voltage proportional to measured quantity. Examples: temperature (LM35, TMP36), light (LDR, phototransistor), distance (ultrasonic), flex sensors.
    • Pros: Provide detailed measurements, can detect gradual changes
    • Cons: Susceptible to noise, requires ADC, needs calibration
    • Use: When you need precise measurements or continuous monitoring
  • Digital Communication Sensors: Use I2C, SPI, or UART protocols. Examples: accelerometers, gyroscopes, environmental sensors (BME280).
    • Pros: Multiple sensors on few wires, built-in processing, high precision
    • Cons: More complex code, requires libraries
    • Use: When you need multiple sensors or high-precision measurements

2. Interfacing Sensors

Different sensors require different interface circuits:

  • Voltage Divider: For resistive sensors (LDR, flex sensor, thermistor). Two resistors create a voltage proportional to resistance.
  • Pull-up/Pull-down Resistors: For digital sensors to ensure clean HIGH/LOW signals. Usually 10kΩ.
  • Amplifiers: For weak signals (like from thermocouples). Op-amps boost signal before ADC.
  • Current Limiting Resistors: For sensors that need specific current (like LEDs used as light sensors).
  • Filtering Capacitors: Reduce noise in analog signals. 100nF ceramic capacitor near sensor helps.

3. Reading Sensor Data

In Arduino, you read sensors using:

  • digitalRead(pin): Returns HIGH or LOW for digital sensors
  • analogRead(pin): Returns 0-1023 (10-bit ADC) or 0-4095 (12-bit ADC) for analog sensors
  • Wire library: For I2C sensors (like BME280, MPU6050)
  • SPI library: For SPI sensors

Important: Always check sensor datasheets for voltage requirements, pin connections, and communication protocols!

4. Calibration and Environmental Considerations

In African environments, sensors face unique challenges:

  • Temperature: High ambient temperatures can affect sensor accuracy. Some sensors need temperature compensation.
  • Dust and Humidity: Can interfere with optical sensors (light, distance). Protect sensors with enclosures.
  • Power Variability: Voltage fluctuations can affect analog readings. Use voltage regulators and filtering.
  • Calibration: Always calibrate sensors in the environment where they'll be used. Take multiple readings and average them.

Mini-Lab: Temperature and Light Monitoring System

Let's build a simple monitoring system that reads temperature and light levels, perfect for agricultural or environmental applications.

Step 1: Gather Components

  • Microcontroller board (Arduino Uno, ATmega328P, or your custom board)
  • Temperature sensor (TMP36 or LM35 - analog output)
  • Light sensor (LDR/photoresistor - analog output)
  • Two 10kΩ resistors (for voltage dividers)
  • 100nF capacitor (for filtering)
  • Breadboard and jumper wires

Step 2: Wire the Circuit

  1. Temperature Sensor (TMP36):
    • VCC → 5V (or 3.3V)
    • GND → GND
    • Output → Analog pin A0
    • Add 100nF capacitor between VCC and GND near sensor
  2. Light Sensor (LDR):
    • One leg → 5V
    • Other leg → 10kΩ resistor → GND
    • Connection point (between LDR and resistor) → Analog pin A1

Step 3: Write the Code

// Pin definitions
#define TEMP_PIN A0
#define LIGHT_PIN A1

void setup() {
  Serial.begin(9600);
  Serial.println("Temperature and Light Monitor");
  Serial.println("-----------------------------");
}

void loop() {
  // Read temperature sensor (TMP36: 10mV per degree C, 500mV offset)
  int tempRaw = analogRead(TEMP_PIN);
  float voltage = (tempRaw / 1024.0) * 5.0;  // Convert to voltage
  float temperature = (voltage - 0.5) * 100;  // Convert to Celsius
  
  // Read light sensor (0 = dark, 1023 = bright)
  int lightRaw = analogRead(LIGHT_PIN);
  float lightPercent = (lightRaw / 1024.0) * 100;  // Convert to percentage
  
  // Print readings
  Serial.print("Temperature: ");
  Serial.print(temperature);
  Serial.print("°C | Light: ");
  Serial.print(lightPercent);
  Serial.println("%");
  
  // Wait before next reading
  delay(1000);
}

Step 4: Test and Calibrate

  1. Upload code and open Serial Monitor
  2. Compare temperature reading with a known thermometer
  3. Adjust calibration if needed (add offset, scale factor)
  4. Test light sensor by covering/uncovering it
  5. Document the range of values you see in your environment

Group Assignment (Optional but Recommended)

As a UNIPOD team, compare multiple sensors:

  • Test at least 3 different sensors (e.g., different temperature sensors, or temperature + light + distance)
  • Record and compare:
    • Noise levels (how much do readings fluctuate?)
    • Range (minimum and maximum values)
    • Stability (do readings drift over time?)
    • Response time (how quickly do they react to changes?)
  • Test in your local environment (account for heat, dust, power variability)
  • Create a shared document comparing sensor performance and cost/availability in your region

Tips for Success

  • Start with simple sensors: Begin with basic analog sensors (temperature, light) before trying complex I2C sensors.
  • Read the datasheet: Always check sensor datasheet for voltage requirements, pin connections, and output characteristics.
  • Use filtering: Add capacitors near sensors to reduce noise. Average multiple readings for stability.
  • Calibrate in your environment: Sensor readings depend on environment. Calibrate where the sensor will actually be used.
  • Test thoroughly: Test sensors in different conditions (day/night, hot/cool, indoor/outdoor) to understand their behavior.
  • Document everything: Record sensor readings, calibration values, and environmental conditions. This data is valuable!
  • Consider power consumption: Some sensors draw significant current. Plan power supply accordingly.
  • Protect sensors: In dusty or humid environments, protect sensors with enclosures or covers.
